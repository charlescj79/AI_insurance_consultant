### 对接方案A：Our MCP Server → Dify as Client (⭐推荐首选)

#### 步骤1: 在Dify中添加MCP Server
```json
// Dify Agent配置 > External Tools > Add MCP Server
{
  "name": "HK Insurance Sales MCP",
  "server_url": "https://your-dify-instance/api/mcp/xxxxx", 
  // 如果是our MCP server作为HTTP端点: stdio模式需通过本地代理
  // Dify Cloud不支持直接stdio → 需Cloud Run部署or ngrok隧道
}
```

#### 步骤2: Our MCP Server的HTTP模式对接Dify
```bash
# 如果MCP server用streamable-http transport启动:
python3 knowledge/sales-skills/mcp/server.py --http-port 8080

# Dify Cloud通过API gateway连接 → Cloud Run部署到香港节点
```

#### 步骤3: 在Dify Agent中映射11个保险工具
- Dify会自动发现our MCP server暴露的所有tools（基于server-card.json的tools列表）
- 每个工具可在Dify Workflow中作为独立node调用
- **关键**: compliance_check必须在所有咨询输出前作为必选前置节点

#### 步骤4: RAG知识库集成
- Dify的RAG引擎是我们的知识载体 → KB文件(kb-status验证过4个文件存在)
- 将保险产品资料放入Dify Knowledge Base → MCP通过compliance_check后输出

### 对接方案B：Our MCP Server ← Dify as Server (反向暴露)

#### 步骤: 将Dify应用发布为MCP Server
```
Dify控制台 > App Settings > MCP Server Tab > Toggle ON
→ Dify自动生成唯一MCP URL（含认证凭证）
→ 外部工具(如Claude Desktop)可连接此Dify app as tool
```

**价值场景**: 将our保险咨询流程封装为Dify workflow → 发布为MCP endpoint → 供多平台复用
